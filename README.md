# CarPulse

MVP сервиса мониторинга автомобильных площадок. Пользователь создаёт радары вручную в личном кабинете, видит свежие совпадения в общей ленте и сможет подключить Telegram для мгновенных уведомлений.

## Что уже есть

- адаптивный личный кабинет;
- форма создания радара без ссылок на готовый поиск;
- реальные радары и фильтры в Cloudflare D1;
- таблицы объявлений, совпадений, запусков источников и очереди уведомлений;
- лента найденных автомобилей из D1;
- экран подключения Telegram по одноразовому коду;
- Auto24-адаптер и отдельный Chromium-сборщик для сервера со статическим IP;
- Cron Trigger `*/30 * * * *`: одна проверка каждые 30 минут;
- контроль свежести внешнего сборщика: ошибка после 50 минут без результата;
- дедупликация объявлений по `source + external_id`;
- единый интерфейс адаптеров Auto24, SS.lv, Nettiauto и mobile.de;
- защищённый ручной запуск `POST /api/sources/auto24/sync`.

Основной production-вариант Auto24 — контейнер `collector/` на Linux-сервере со
статическим IP. Он получает из CarPulse активные радары, запускает обычный
Chromium с постоянным профилем, собирает точную выдачу каждого набора фильтров и
отправляет записи обратно через закрытый endpoint. При явной защитной странице,
`403` или `429` запуск записывается как `blocked`; при остановке контейнера
CarPulse показывает «Сборщик не отвечает». SS.lv, Nettiauto и mobile.de пока не
подключены.

## Поток данных

```text
Linux static IP → Chromium → Auto24 → CarPulse collector API → D1 listings
                                                        → radar_matches
                                                        → notification_deliveries

Cloudflare Cron → контроль результата не старше 50 минут
```

## Запуск

```bash
npm install
npm run dev
```

Локальный запуск поднимает приложение и отдельный планировщик. Он делает первую
проверку Auto24 после старта, а затем вызывает Worker в `:00` и `:30`. Если
приложение и планировщик нужно запустить раздельно:

```bash
npm run dev:server
npm run dev:cron -- --now
```

Проверка production-сборки:

```bash
npm run build
npm test
npm run lint
```

После изменения `db/schema.ts`:

```bash
npm run db:generate
```

Для ручной синхронизации в Cloudflare нужно добавить секрет `AUTO24_SYNC_SECRET` и вызвать:

```bash
curl -X POST https://<worker>/api/sources/auto24/sync \
  -H "Authorization: Bearer <AUTO24_SYNC_SECRET>"
```

## Внешний сборщик Auto24

Инструкция установки и переменные окружения находятся в
[`collector/README.md`](collector/README.md). После первого успешного запуска
нужно задать production-переменные CarPulse:

```text
AUTO24_COLLECTOR_SECRET=<общий длинный случайный секрет>
AUTO24_MODE=external
```

`AUTO24_MODE=external` следует включать только после запуска контейнера, чтобы
до переключения продолжал работать прежний Browser Run.
