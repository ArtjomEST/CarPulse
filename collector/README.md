# CarPulse Auto24 collector

Отдельный сборщик для сервера со статическим IP. Он получает активные радары
из CarPulse, открывает Auto24 в обычном Chromium с постоянным профилем и
возвращает найденные объявления в защищённый endpoint.

## Требования

- Linux-сервер со статическим публичным IP;
- Docker и Docker Compose;
- один общий секрет `AUTO24_COLLECTOR_SECRET` в CarPulse и на сервере.
- `CARPULSE_SITES_BYPASS_TOKEN`, пока кабинет закрыт авторизацией Sites.

## Запуск

```bash
cp .env.example .env
docker compose up -d --build
docker compose logs -f auto24-collector
```

Контейнер запускает проверку сразу после старта, затем в начале каждого
30-минутного интервала. Профиль Chromium хранится в Docker volume и сохраняет
cookies между запусками.

`CARPULSE_SITES_BYPASS_TOKEN` передаётся только как секрет окружения и позволяет
контейнеру обратиться к collector API закрытого кабинета. Его нельзя добавлять в
репозиторий или выводить в логи.

CarPulse продолжает контролировать расписание своим 30-минутным cron: если
сборщик не прислал ни одного результата больше 50 минут, площадка получит
состояние «Сборщик не отвечает». Контейнер перезапускается автоматически после
сбоя процесса.

Для одной диагностической проверки без постоянного цикла:

```bash
docker compose run --rm auto24-collector node src/index.mjs --once
```
